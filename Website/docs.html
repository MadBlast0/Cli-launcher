Getting Started

CLI Launcher is an open-source desktop panel for terminal AI coding agents. It installs, updates, and launches every agent from a single place — then gives you one-click launch, update, and management for every tool.

The fastest way to try it is the one-line installer. Pick your platform below, or grab a native installer from the Downloads page.

curl -fsSL https://cli-launcher.veyl.in/install.sh | sh

Installation

Choose whichever method fits your workflow. The script and PowerShell installers are mirrored at cli-launcher.veyl.in/install.sh and cli-launcher.veyl.in/install.ps1.

curl · macOS / Linux
curl -fsSL https://cli-launcher.veyl.in/install.sh | sh
wget · macOS / Linux
wget -qO- https://cli-launcher.veyl.in/install.sh | sh
PowerShell · Windows
powershell -ExecutionPolicy Bypass -c "irm https://cli-launcher.veyl.in/install.ps1 | iex"
From source · all platforms
git clone https://github.com/MadBlast0/Cli-launcher && cd Cli-launcher && npm install && npm run build

Prefer a graphical setup? Download the signed .dmg, .msi, or .AppImage from the Downloads page.

Supported Agents

CLI Launcher manages 32+ agents across three runtimes: 18 Node-based, 6 Python-based, and 8 standalone binaries. Every entry ships with its real package string so you can verify it against npm or pip.

Notable agents include Claude Code, OpenCode, and Gemini CLI — the full, filterable list lives on the homepage inventory.

Agents are managed side by side, so you can install, update, or uninstall any of them from the panel without leaving its interface — no switching terminals or juggling global environments by hand.

YOLO Mode

YOLO Mode flips one switch to launch supported agents with their auto-approve flags enabled — no more permission prompts mid-task. This is ideal for unattended, fully autonomous coding runs. CLI Launcher injects the correct flag per agent, so you never hand-edit a command.

Example injected flag (Claude Code)
claude --dangerously-skip-permissions

Toggle YOLO Mode from the panel for any supported agent. Because it auto-approves every action, use it only in trusted working directories.

Theming

CLI Launcher ships with light and dark themes. Use the toggle in the top bar to switch instantly; your choice is persisted locally and the app also respects your system preference on first launch.

Colors are driven by CSS variables (see style.css), so the panel, docs, and marketing site all share one consistent, flat design language.

Troubleshooting

Command not found after install
The launcher was added to your PATH but your shell session may be stale. Restart the terminal (or run source ~/.zshrc / source ~/.bashrc) and try again.
Install script blocked on Windows
PowerShell may restrict remote scripts. Re-run the installer with -ExecutionPolicy Bypass as shown in the Installation section.
Managing agents from the panel
Each agent is installed and updated through CLI Launcher. If a tool is missing, install it through the panel rather than your global environment.
Still stuck?
Open an issue on GitHub — the project is open-source under the MIT License.

Frequently Asked Questions

How do I install CLI Launcher?

Run the one-line installer — curl -fsSL https://cli-launcher.veyl.in/install.sh | sh on macOS and Linux, or powershell -ExecutionPolicy Bypass -c "irm https://cli-launcher.veyl.in/install.ps1 | iex" on Windows. You can also download a native .dmg, .msi, or .AppImage from the Downloads page, or build from source.

Which terminal AI agents can CLI Launcher manage?

CLI Launcher manages 32+ agents across three runtimes: 18 Node-based (Claude Code, OpenCode, Gemini CLI, GitHub Copilot, Codex, and more), 6 Python-based (Aider, Open Interpreter, Shell-GPT, gptme, RA.Aid, OpenHands), and 8 standalone binaries (Goose, Cursor, Amazon Q, Plandex, Mods, aichat, Fabric, and others).

What is YOLO Mode and is it safe?

YOLO Mode launches supported agents with their auto-approve flags enabled (for example, Claude Code's --dangerously-skip-permissions) so they run unattended without permission prompts. Because it auto-approves every action, use it only in trusted working directories.

Does CLI Launcher sandbox the agents it installs?

No. CLI Launcher is a management panel, not a sandbox. It installs and launches agents using your system's existing Node, Python, or standalone runtimes; it does not create isolated environments or virtualize dependencies.

Is CLI Launcher free and open source?

Yes. CLI Launcher is MIT-licensed and free to use. The source code is available on GitHub.
